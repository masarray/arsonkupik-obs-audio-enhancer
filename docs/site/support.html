Where support goes

Funding the work users normally do not see

Open-source audio quality depends on more than writing DSP code. Reliable public releases require repeated platform builds, installation checks, listening comparisons, regression tests, documentation, and investigation across different sources and sample rates.

Release engineeringWindows installer, portable package, Linux archive, CI, and GitHub Release maintenance
Audio-quality validationCrackle, pumping, gain-staging, stereo, clipping, and macro-response testing
OBS compatibilityBuild-system updates, plugin packaging, installation paths, and version checks
DocumentationPublic guides, issue forms, release notes, support material, and landing-page maintenance
Funding principles

Support never creates a paywall

Same public product

Donations do not unlock exclusive DSP features, private presets, or a closed premium binary.

Independent engineering

Sponsors may suggest work, but audio quality, architecture, security, licensing, and release decisions remain independent.

Optional recognition

Supporters may opt into recognition in the public supporters file. Anonymous support is fully respected.

No guaranteed deliverable

A donation is support for the project, not payment for a deadline, custom feature, or priority bug fix.
